# Структура проекту

Цей репозиторій використовується для створення CCP-проектів. Нижче наведено структуру каталогів та опис того, для чого використовується кожна папка та файл.

```plaintext
uk/
├── код/
│ ├── назва-проєкту-повна/
│ │ ├── main.py
│ │ └── project_config.yml
│ └── назва-проєкту-стартер/
│ ├── main.py
│ └── project_config.yml
├── зображення/
│ └── .keep
├── ресурси/
│ └── .keep
├── рішення/
│ └── .keep
├── README.md
├── meta.yml
├── step_1.md
├── step_2.md
└── step_3.md
```

## Описи каталогів та файлів

### `код/`
Містить код Python або HTML, який використовується в редакторі коду Raspberry Pi.
- Кожен проект має бути у своєму окремому підкаталозі. Назви каталогів мають бути описовими, але в іншому гнучкими.
- Усередині кожного підкаталогу:
  - `main.py` (для проектів Python) або `index.html` (для проектів HTML)
  - `project_config.yml`, що містить:
    - `name`: Назва проєкту
    - `ідентифікатор`: Унікальний ідентифікатор проєкту
    - `type`: 'python' або 'html'
    - `build`: `false` за замовчуванням; встановити значення `true` для збірки проекту редактора

### `зображення/`
Зберігає всі зображення для проєкту.
- Повинен містити `banner.png` (головне зображення проєкту).
- Відредаговані зображення повинні супроводжуватися їх оригінальними версіями.
- Додаткові допоміжні зображення слід називати розумно.

### `ресурси/`
Містить допоміжні ресурси для проєкту.
- Обов'язково додайте PDF-версію проекту з чіткою назвою.
- Сюди також можна додавати інші ресурси, такі як офлайн-проєкти для початку роботи, ресурси або допоміжні файли.
- Учні можуть завантажити `.zip` цієї папки за посиланням:  
  `https://rpf.io/p/uk-UA/project-slug-go`

### `рішення/`
Містить остаточну завершену версію проєкту, якщо вона потрібна.
- Учні можуть завантажити це за посиланням:  
  `https://rpf.io/p/uk-UA/project-slug-get`

### `meta.yml`
Визначає метадані проєкту, що використовуються для публікації. Приклади полів:
```yaml
title: Назва проекту (з регістру речень)
hero_image: images/banner.png
description: Короткий опис проекту
listed: false
pdf: false
steps:
  - title: Що ви зробите
  - title: Назва кроку
    completion:
      - angažed
  - title: Назва кроку
    completion:
      - внутрішній      
  - назва: Виклик
    виклик: правда
    завершення:
      - зовнішній
```

- **`у списку`**:
  - `true`: проєкт опубліковано з гілки master
  - `false`: проект не є публічно опублікованим (але все ще доступний через URL-адресу слуга)
- **`pdf`**: Шлях до PDF-файлу в `resources/`, що робить його доступним для завантаження
- **`завершення`**: Метрики прогресу учня
  - `залучений`: Учень досяг першого кроку "творення"
  - `internal`: Заключний етап "створення" проекту
  - `зовнішній`: останній крок проекту, часто складний
- **Виклики**: Необов'язково, надайте додаткові завдання та враховуйте їх як «зовнішні» кроки виконання

### `README.md`
Документація до проєкту, включаючи структуру та пояснення змісту.

### `step_1.md`, `step_2.md`, `step_3.md`
Файли Markdown, що містять покрокові інструкції для учнів.
- Кожен крок відповідає розділу навчального посібника.
- Кількість кроків залежить від проекту.
